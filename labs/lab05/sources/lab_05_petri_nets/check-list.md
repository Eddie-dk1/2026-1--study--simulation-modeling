При составлении лабораторной работы следует придерживаться определённой структуры. Здесь рассматриваются лабораторные работы, выполняемые на компьютере.


Основные положения
Рабочий каталог лабораторной работы должен иметь унифицированную структуру.
Исходный код программ, отчёты по лабораторным работам и т.д. размещаются в системе контроля версий git.
Выполнение лабораторной работы документируется в виде скринкаста.
Должны выполняться соглашения об именовании.
При проверке лабораторных работ следует использовать понятные критерии оценки.

Критерии оценки:
1.Все элементы, все скринкасты, все хостинги
2.markdown и все производные форматы
3.Все этапы, структурировано
4.Все элементы поименованы, есть ссылки, библиография, корректный bib-файл
5.git + релизы + общепринятые коммиты + семантические версии + changelog
6.Везде выполнено соглашение об именовании
7.Выполнено в полном объёме, без ошибок
